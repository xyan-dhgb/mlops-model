🔗 Phân Tích Chiến Lược Hợp Nhất Muộn

📍 Vị Trí Hợp Nhất: Xử Lý Phía Server

Quá trình hợp nhất diễn ra trên server sau khi cả hai đặc trưng từ bảng và hình ảnh đã được trích xuất và xử lý qua các nhánh mạng thần kinh tương ứng.

Aspect Client-Side Fusion Server-Side Fusion (Current)
Performance Limited by device capabilities High-performance GPU/CPU clusters
Latency No network transfer for fusion Network latency for data transfer
Security Sensitive medical data on device Centralized security controls
Model Updates Requires app updates Real-time model deployment
Scalability Device-dependent Easy horizontal scaling
Privacy Data stays on device Data transmitted to server

✅ Why Server-Side Fusion is Optimal for This System:

  • Medical Grade Accuracy: Requires powerful hardware for reliable ResNet50 inference
  • Regulatory Compliance: Centralized logging and audit trails for medical applications
  • Model Complexity: 160-dimensional fusion layer needs substantial computational resources
  • Continuous Learning: Server can accumulate and learn from new cases
  • Expert Validation: Allows for human expert review and feedback integration

⚙️ Implementation Details

🔗 Fusion Layer Implementation

# Fusion Layer in build_multimodal_model() combined = concatenate([x_tab.output, x_img.output]) # Post-fusion processing x = Dense(64, activation='relu')(combined) x = Dropout(0.3)(x) x = Dense(32, activation='relu')(x) output = Dense(num_classes, activation='softmax')(x)

📊 Feature Extraction Summary

  • Tabular Features: 32-dimensional vector from clinical metadata
  • Image Features: 128-dimensional vector from ResNet50 + Dense layer
  • Fused Features: 160-dimensional concatenated vector
  • Final Classification: 3-class output (Normal, Benign, Malignant)
